Canvas 2D text rendering

HTML rich text,
drawn on canvas.

render-tag takes HTML + CSS and draws it onto a canvas element with the 2D API. No SVG. No foreignObject. Synchronous. Zero dependencies.

By design, render-tag focuses on rich text only — paragraphs, headings, lists, tables, inline formatting. It is not designed for interactive elements like buttons, inputs, iframes, or complex layouts. This focus is what makes it fast.

API

      

      
render(config) → { canvas, height, layoutRoot, lines }

      

        

          
            
OptionTypeDefaultDescription

          
          
            
htmlstring—HTML string to render. Include <style> tags for CSS.

            
widthnumber—Layout width in CSS pixels. Required.

            
heightnumberautoFixed height. Auto-sized from content if omitted.

            
ctxCanvasRenderingContext2D—Existing context to draw onto (no resizing/scaling).

            
canvasHTMLCanvasElement | OffscreenCanvascreatedTarget canvas (mutually exclusive with ctx).

            
pixelRationumberdevicePixelRatioDevice pixel ratio for HiDPI.

            
accuracy'balanced' | 'performance''balanced'DOM probes for cross-browser line height accuracy.

          
        

      

      
Synchronous. Fonts must be loaded before calling — use document.fonts.load() or document.fonts.ready. Re-render when fonts load if needed.


      
Advanced: layout once, draw many

      
Split layout and rendering to draw the same content onto multiple targets:

      
layout(config) → { layoutRoot, height, lines }

      
drawLayout(config) → { canvas }

    

  


  
  

    

      

        
import { render } from 'render-tag';

const { canvas, height } = render({
  html: '<p>Hello <strong>world</strong></p>',
  width: 400,
});

document.body.appendChild(canvas);
